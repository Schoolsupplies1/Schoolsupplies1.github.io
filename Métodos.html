Método grafico

El método gráfico es una técnica que se utiliza para resolver problemas matemáticos, especialmente aquellos relacionados con la optimización. Consiste en representar gráficamente las restricciones y la función objetivo en un plano cartesiano. Al observar el gráfico, se busca el punto donde las restricciones se intersecan y determinan el mejor resultado posible según el objetivo establecido. Es una forma visual y sencilla de encontrar soluciones aproximadas a problemas matemáticos.

Las combinaciones requeridas para formar cada paquete son las siguientes:

Paquete B: Requiere 1 lápiz, 2 cuadernos y 2 plumas.

Proceso

Utilidad máxima Z = 10x + 15y (ganancia total)
Sujeto a:
Disponibilidad de lápices: 2x + y ≤ 2000 
Disponibilidad de cuadernos: x + 2y ≤ 1500 
Disponibilidad de plumas: x + 2y ≤ 1000
x, y ≥ 0   

Para hacer la resolución de las restricciones debemos convertir las desigualdades a igualdades:
2x + y ≤ 2000 → 2x + y = 2000 
x + 2y ≤ 1500 → x + 2y = 1500
x + 2y ≤ 1000 → x + 2y = 1000 

El siguiente paso es despejar (x, y) en cada restricción: 
1.     Restricción de disponibilidad de lápices: 2x + y = 2000 
Sustituimos y por 0 y resolvemos para x: 
2x + 0 = 2000 
2x = 2000 
x = 1000 
Sustituimos x por 0 y resolvemos para y: 
2(0) + y = 2000
y = 2000
Obtenemos los puntos (1000, 0) y (0, 2000) 

2.     Restricción de disponibilidad de cuadernos: x + 2y = 1500
Sustituimos y por 0 y resolvemos para x:
x + 2(0) = 1500 
x = 1500 
Sustituimos x por 0 y resolvemos para y:
0 + 2y = 1500 
y = 750 
Obtenemos los puntos (1500, 0) y (0, 750)

3.     Restricción de disponibilidad de plumas: x + 2y = 1000   
Sustituimos y por 0 y resolvemos para x: 
x + 2(0) = 1000 
x = 1000 
Sustituimos x por 0 y resolvemos para y: 
0 + 2y = 1000 
y = 500 
Obtenemos los puntos (1000, 0) y (0, 500) 
A continuación, graficamos los puntos obtenidos en un plano cartesiano:
A: (1000, 0) y (0, 2000) 
B: (1500, 0) y (0, 750) 
C: (1000, 0) y (0, 500) 

Región factible:

Para determinar la región factible debemos seleccionar aleatoriamente dos puntos, punto alto (por encima de la recta) y punto bajo (por debajo de la recta), para después crear las condiciones en base a las restricciones:   
A)    Disponibilidad de lápices: 2x + y ≤ 2000 
Punto alto (2100,2100) se van a sustituir los valores de (x, y) por los de el punto. 
2(2100) + 2100 = 4200 + 2100 = 6300 
Al no respetar la condición (≤ 2000) todos los valores por encima de la recta A serán desestimados

Punto bajo (500, 500) 
2(500) + 500 = 1000 + 500 = 1500 
Respeta la condición ya que es menor a 2000 y también cercano a la cifra.

B)    Disponibilidad de cuadernos: x + 2y ≤ 1500 
Punto alto (1000,1000) 
1000+2(1000) =1000+ 2000= 3000 
No respeta la condición, por lo que se desestiman los puntos por encima de la recta B. 

Punto bajo (400,400) 
400+2(400) =400+800 = 1200
Respeta la condición ya que es menor a 1500 y también cercano a la cifra.

C)    Disponibilidad de plumas: x + 2y ≤ 1000 
Punto alto (600,200) 
600+2 (200) =600+400= 1000 
Respeta la condición y da la cantidad exacta, por lo que, no se desestimaran los puntos por encima de la recta C.

Punto bajo (400,300) 
400+ 2(300) =400+600= 1000 
También respeta la condición por lo que se considera que los puntos alto y bajo de la recta C son parte de la región factible.

Región factible: Para determinar la región factible primero debemos seleccionar los puntos de intersección que existen entre las restricciones. 

(0, 750) 
(0,500) 
(300,850) 
(1000, 0) 
Después, debemos multiplicar los puntos por la Utilidad máxima Z = 10x + 15y 
(0, 750) = 10(0) + 15(750) = 11,250 
(0,500) = 10(0) + 15(500) = 7, 500 
(300,850) = 10(300) + 15(850) = 15, 750 
(1000, 0) = 10(1000) + 15(0) = 10, 000 
En base a los cálculos anteriores se escribe la siguiente interpretación: Para que la empresa School Supplies maximice su beneficio total debe elaborar 300 paquetes del tipo A y 850 paquetes del tipo B, lo que generaría una ganancia de $15, 750. 

Método de costos mínimos 

El método de costos mínimos consiste en hacer las cosas de la manera más barata posible. Si tienes un negocio y quieres fabricar algo, debes encontrar la forma de gastar la menor cantidad de dinero. Esto implica producir la cantidad justa de productos, ni más ni menos, para evitar gastos innecesarios.

Método húngaro

El Método Húngaro, o algoritmo de asignación de Munkres, es una técnica utilizada para resolver problemas de asignación. Se aplica cuando hay tareas que deben ser realizadas por personas o recursos, y se busca la asignación óptima que minimice el costo total o maximice el beneficio.

Método Vogel

El Método de Vogel es una técnica utilizada en la asignación de recursos para minimizar los costos de transporte. Se basa en la diferencia entre los dos costos más bajos en cada fila y columna para determinar las asignaciones óptimas. Ayuda a lograr soluciones más equilibradas y con menores costos, pero puede requerir ajustes adicionales para alcanzar una solución óptima.

Modelo de planeación de proyector (Critical PATH method CPM) 

El CPM es un modelo de planificación de proyectos que utiliza la identificación del camino crítico para determinar la duración total del proyecto y gestionar las actividades de manera eficiente. Es una técnica valiosa para programar, controlar y ajustar proyectos en función de las interdependencias y restricciones de las actividades.

Método MODI

El método MODI (Modified Distribution) es un algoritmo utilizado en la programación lineal para encontrar una solución óptima en problemas de transporte y asignación. Se utiliza para determinar los costos marginales de asignar unidades adicionales de recursos a diferentes actividades, y así identificar la variable no básica que ingresará a la solución básica óptima. El método MODI involucra iteraciones donde se calculan los costos marginales y se ajustan las variables básicas hasta que se alcanza la solución óptima.

Método cruce del arroyo 

El método del cruce del arroyo, también conocido como método de la corriente de agua o método del río, es una técnica utilizada en la planificación de proyectos para determinar la secuencia más eficiente de actividades. Este método se basa en la analogía de cruzar un arroyo, donde cada actividad se representa como una piedra en el camino y el flujo del agua representa el flujo de tiempo en el proyecto.

Algoritmo de Dijkstra

El algoritmo de Dijkstra es un algoritmo de búsqueda de caminos mínimos en un grafo dirigido y ponderado. Su objetivo es encontrar la ruta más corta desde un nodo de origen a todos los demás nodos del grafo. Utiliza una estrategia de exploración iterativa, actualizando las distancias mínimas conocidas a medida que se recorren los nodos adyacentes. Al finalizar, se obtiene la distancia mínima desde el nodo de origen a cada uno de los demás nodos, así como el camino óptimo para llegar a cada uno de ellos.
